# QA Case Study - Exploratory & Usability Testing

## Overview
this repository contains a QA case study based on a recorded user session. The purpose of this assessment was to identify bugs, usability issues, and unclear system behavior from a real user's perspective.

No proprietary, confidential, or internal company materials are included.

## Testing Scope
- Testing Type: Exploratory / Usability Testing
- Platform: Web application
- Environment: Desktop browser
- Focus: User feedback, system errors, unclear UI responses

## Skills Demonstrated
- Exploratory testing
- Bug identification and documentation
- UX and usability analysis
- Clear written communication
- Expected vs Actual behavior reporting
